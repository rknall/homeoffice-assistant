   - Border: Dashed 1px border (to distinguish from HomeOffice events)

3. **Todos with Due Dates**
   - Icon: â˜‘ (checkbox)

  - Todo: Full title, due date, status
- **Click:** Open appropriate detail view (see Modal Mockups section)
- **Multi-day events:** Span across multiple day columns

**Visual Density:**
- Compact view for small time slots
- Automatic text truncation with ellipsis (...)
- Overlapping events displayed side-by-side or stacked

   - Top-left corner
   - Bold font
   - Current day highlighted (different background)
   - Grayed out for dates outside current month

2. **Event Indicators:**
   - **Multi-day events**: Horizontal bar spanning cells
   - **Single events**: Colored dot (â—) + truncated name
   - **Todos**: Checkbox icon (â˜‘) + truncated name
   - Max 3 visible items per cell

   - "+N more" text at bottom
   - Clickable to expand day view or show popover

**Interactions:**
- **Click day number**: Expand day view (show all events for that day)
- **Click event**: Open appropriate detail modal
- **Click "+N more"**: Show popover with all events for that day
- **Hover event**: Show tooltip with basic info

**Visual Styling:**
- **Current day**: Light blue background
- **Today**: Bold border or distinct highlight
- **Weekend days**: Light gray background
- **Events**: Use same color scheme as week view

### Connected Calendar Card
Each connected calendar shows:
- **Calendar icon** and **name**
- **Color picker** for calendar color
- **Status indicator:** Active/Syncing/Error
- **Last sync timestamp**
- **Actions:**
  - `[Change Color]` - Opens color picker
  - `[Disconnect]` - Removes calendar connection (with confirmation)

### Connect Button Flow
1. Click `[+ Connect Google Calendar]`
2. OAuth flow initiates (new window/tab)
3. User authorizes Google account
4. Google calendar selection screen
5. User selects calendar from list
6. Confirmation and return to settings
7. New calendar card appears
8. Initial sync begins

## 11. Implementation Notes

### Calendar Library Recommendations
Based on the mockups, recommended libraries:

**Option 1: FullCalendar** (Recommended)
- Pros: Feature-rich, excellent week/month views, good customization
- Cons: Larger bundle size
- URL: https://fullcalendar.io/

**Option 2: React Big Calendar**
- Pros: Lightweight, React-native, flexible
- Cons: Less polished out-of-box, more custom styling needed
- URL: https://jquense.github.io/react-big-calendar/

**Option 3: Custom with date-fns**
- Pros: Full control, minimal dependencies
- Cons: More development time, need to implement all features
- Use if: Bundle size is critical or need very specific behavior
